# shared/

Cross-workspace generated artifacts. This is a plain directory at the repo root, **not** an npm workspace — the frontend imports the type file via relative path (`../../shared/openapi.d.ts`) rather than a package name. We picked this over adding a third workspace because the only files here today are two generated artifacts (`openapi.json`, `openapi.d.ts`) and there's no build, no tests, no public API surface to maintain. A workspace would be overhead without payoff; revisit if shared runtime code (e.g. Zod schemas) ever lands here.

## Files

- `openapi.json` — OpenAPI 3 document, generated by `npm run -w backend openapi:dump` from the live NestJS controller decorators.
- `openapi.d.ts` — TypeScript types generated by `openapi-typescript` from `openapi.json`. Frontend code imports `paths` / `components` from here.

## Regenerating

From the repo root:

```bash
npm run openapi          # dump + codegen
npm run openapi:dump     # backend → shared/openapi.json only
npm run openapi:codegen  # shared/openapi.json → shared/openapi.d.ts only
```

Both files are checked into git so the frontend can typecheck without running the backend first.
